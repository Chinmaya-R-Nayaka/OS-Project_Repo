System Calls Implemented:
1. getprocinfo()


Description:
This project implements custom system calls in the xv6 operating system

1. getprocinfo() --> System Call
	The getprocinfo() system call is implemented in the xv6 operating system to retrieve information about 
currently active processes. It accesses the kernel’s process table and displays details such as the Process ID (PID)
and the state of each process. This system call demonstrates how user programs interact with the Kernel to obtain 
system level information. The System Call is invoked from user program and executed in kenel through 
function sys_getprocinfo()


2.shmget() and shmat() 


Description:
2.i. shmget() -->System Call
        The shmget() system call acts as the allocator for a persistent physical memory segment that can be shared 
        across the entire system. In our implementation, the kernel checks if a global physical page has already 
        been allocated; if not, it calls kalloc() to reserve a page of RAM and returns a unique identifier 
        to the user. 
2.ii. shmat() -->System Call
        The shmat() system call is responsible for "attaching" the previously allocated physical page to a specific 
        process's virtual address space. It calculates a free virtual address at the top of the process's memory and
        uses mappages to link that address to the shared memory physical address, notably tagging the entry with a custom 
        PTE_SHM flag. Using the PTE_SHM flag, we modified uvmcopy so that children automatically inherit the exact same 
        physical mapping during a fork(), allowing seamless communication.
